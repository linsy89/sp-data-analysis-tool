# SP 广告数据分析工具 - 开发指南

## 项目概述

这是一个基于 Streamlit 的 Web 应用，用于 SP 广告数据的维度提取和多维度分析。

- **仓库地址**：https://github.com/linsy89/sp-data-analysis-tool
- **部署平台**：Replit
- **技术栈**：Python 3.11, Streamlit 1.51.0, Pandas 2.3.3

---

## 开发工作流程

### 本地开发（您的电脑）

#### 步骤 1：修改代码
编辑项目文件：
- `app.py` - Streamlit 主应用
- `data_processor.py` - 数据处理模块
- 其他辅助文件

#### 步骤 2：本地测试（可选）
如果本地环境可用：
```bash
streamlit run app.py
```

#### 步骤 3：提交到 Git
```bash
# 查看修改
git status

# 添加所有修改的文件
git add .

# 提交（务必写清楚提交信息）
git commit -m "功能描述：具体做了什么"

# 推送到 GitHub（main 分支）
git push origin main
```

**提交信息规范示例**：
- ✅ `feat: Add single-dimension analysis UI for Step 2`
- ✅ `fix: Replace empty attribute values with '未分类'`
- ✅ `docs: Update development guide`
- ❌ `update` （太模糊）
- ❌ `test` （不清楚）

---

### Replit 部署（云端）

#### 步骤 4：拉取最新代码
在 Replit Shell 执行：
```bash
git pull origin main
```

**注意**：
- ✅ 使用 `git pull`（推荐）
- ❌ 不要使用 `git reset --hard`（会删除虚拟环境配置）

#### 步骤 5：应用自动重新加载

**自动方式**（推荐）：
- Streamlit 会自动检测代码变化
- 应用会自动重新加载（可能需要 5-10 秒）
- 刷新浏览器即可看到新功能

**手动方式**（如果自动加载失败）：
```bash
# 按 Ctrl+C 停止应用
# 重新启动
streamlit run app.py --server.port=5000 --server.address=0.0.0.0
```

## 常见问题

### Q1: Git 拉取时出现 "divergent branches" 错误

**原因**：Replit 本地有修改和 GitHub 远程版本冲突

**解决方案**：
```bash
# 删除冲突的本地文件（通常是 .replit）
rm .replit

# 重新拉取
git pull origin main
```

### Q2: 修改后 Streamlit 没有自动重新加载

**可能原因**：
- 修改的文件较多，自动检测延迟
- 浏览器缓存

**解决方案**：
```bash
# 手动重启应用
# 1. 按 Ctrl+C 停止
# 2. 重新运行
streamlit run app.py --server.port=5000 --server.address=0.0.0.0

# 3. 刷新浏览器（Ctrl+R 或 Cmd+R）
```

### Q3: 我不小心在 Replit 中修改了代码，想恢复

**安全方案**：
```bash
# 查看当前状态
git status

# 放弃本地修改，恢复到 GitHub 版本
git checkout -- app.py  # 恢复单个文件
# 或
git reset --hard origin/main  # 恢复所有文件（谨慎！）
```

### Q4: 如何在 Replit 中测试单元测试

```bash
# 运行所有测试
pytest test_data_processor.py -v

# 运行特定测试类
pytest test_data_processor.py::TestExtractDimensions -v

# 显示代码覆盖率
pytest test_data_processor.py --cov=data_processor
```
